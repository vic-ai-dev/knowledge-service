# 上线Checklist规范

> 参考：腾讯发布规范、阿里云上线流程、Google SRE 变更管理

## 1. 概述

### 1.1 目的
通过标准化的上线检查清单（Checklist），确保每次发布前所有准备工作到位、风险可控、回滚方案完备，降低发布导致的生产事故概率。

### 1.2 适用范围
适用于技术开发部所有系统的正式环境发布，包括功能发布、热修复、配置变更、数据迁移等。

---

## 2. 发布类型与流程

### 2.1 发布类型

| 类型 | 定义 | 审批链 | 执行窗口 |
| --- | --- | --- | --- |
| 常规发布 | 功能迭代、Bug 修复 | 技术负责人 + QA | 每周固定窗口 |
| 紧急发布 | P0/P1 缺陷修复 | 技术负责人 + 架构师 | 紧急窗口（24h） |
| 配置发布 | 配置中心变更 | 配置管理员 | 工作时间内 |
| 数据发布 | 数据订正、数据迁移 | DBA + 技术负责人 | 小黑窗 |

### 2.2 发布流程概览

```
代码冻结 > 测试报告 > 上线审批 > 环境准备 > 数据库变更 > 应用部署 > 灰度验证 > 全量发布 > 观察期
```

---

## 3. 发布前 Checklist

### 3.1 代码检查

- [ ] 代码已通过 Code Review
- [ ] 所有单元测试通过（覆盖率不低于标准）
- [ ] 静态代码扫描无 P0/P1 问题（SonarQube / CodeQL）
- [ ] 安全扫描无高危漏洞（SAST / DAST）
- [ ] 依赖包无已知漏洞（SCA / Dependabot）
- [ ] 代码已合并到目标分支（release / master）
- [ ] Git Tag 已打好（格式：v{主}.{次}.{修订}）

### 3.2 测试验收

- [ ] 功能测试报告签署通过
- [ ] 集成测试通过
- [ ] 回归测试通过（核心用例 100%）
- [ ] 性能压测通过（或评估不影响性能）
- [ ] UAT 验收通过（业务方确认）
- [ ] 兼容性测试通过（浏览器/API 版本/数据库版本）

### 3.3 环境准备

- [ ] 预发布环境（Staging）验证通过
- [ ] 生产环境配置已审核（配置中心 diff）
- [ ] 数据库变更脚本已审批并预执行验证
- [ ] 依赖的外部服务版本已确认（第三方 API / 中间件）
- [ ] SSL 证书未过期
- [ ] CDN 缓存刷新计划已确认
- [ ] 域名/DNS 变更已完成

### 3.4 回滚方案

- [ ] 回滚方案文档已编写
- [ ] 应用回滚步骤已验证（回滚到上一版本的脚本）
- [ ] 数据库回滚脚本已准备并验证
- [ ] 数据回滚触发条件已定义
- [ ] 回滚预估时间已评估（RTO）
- [ ] 回滚后数据有效性已验证

### 3.5 灰度策略

- [ ] 灰度发布方案已定稿
- [ ] 灰度比例和时间计划已确定
- [ ] 灰度验证指标已定义（错误率、RT、业务指标）
- [ ] 灰度回滚条件已明确
- [ ] 灰度期间监控已配置

### 3.6 通知与值班

- [ ] 发布通知已发送（钉钉/邮件/企微）
- [ ] 受影响系统负责人已通知
- [ ] 值班人员已确认（发布期间在线）
- [ ] 客服/运维团队已获知可能影响
- [ ] 业务方已知悉发布时间窗口

## 4. 灰度发布规范

### 4.1 灰度策略模板

```markdown
## 灰度发布方案 - [项目名称] v[版本号]

### 灰度范围
- 灰度比例：[ ]%（建议从 1% 开始）
- 灰度用户：按 [渠道 / 地域 / 用户 ID hash]
- 灰度时长：[ ] 小时/天

### 灰度验证指标
- 错误率：P0 接口错误率 < 0.1%
- P99 RT：低于基准值 1.2 倍
- 业务指标：出单成功率无下降、支付成功率无下降
- 系统资源：CPU < 70%、内存 < 80%

### 灰度升级条件
1. 灰度观察 [ ] 小时无 P0/P1 告警
2. 核心业务指标正常
3. 性能指标在阈值内

### 灰度回滚条件
1. 错误率超过 [ ]%
2. P0 接口不可用
3. 关键业务指标显著下降（如 10%）
4. 出现资损
```

### 4.2 灰度到全量

```
灰度稳定运行 > 扩灰度比例 > 稳定观察 > 全量 > 持续观察 30 分钟
```

| 步骤 | 操作 | 验证 | 持续时间 |
| --- | --- | --- | --- |
| 1 | 灰度 1% | 验证指标正常 | 30 分钟 |
| 2 | 灰度 5% | 验证指标正常 | 30 分钟 |
| 3 | 灰度 20% | 验证指标正常 | 1 小时 |
| 4 | 灰度 50% | 验证指标正常 | 1 小时 |
| 5 | 全量 100% | 验证指标正常 | 30 分钟观察 |

## 5. 发布中 Checklist

### 5.1 数据库变更执行

- [ ] 备份受影响表
- [ ] 先执行回滚脚本验证（在某台从库执行）
- [ ] 按顺序执行 DDL（从依赖少的表开始）
- [ ] 每一步执行后验证（`SHOW TABLES`、`DESC`）
- [ ] DML 数据变更先做 COUNT 预览

### 5.2 应用部署

- [ ] 停止旧版本流量（摘流量）
- [ ] 部署新版本（滚动更新 / 蓝绿部署）
- [ ] 启动后健康检查通过（`/health` / `/actuator/health`）
- [ ] 加入流量观察
- [ ] 监控面板确认指标正常

### 5.3 发布确认

- [ ] 发布系统显示状态正常
- [ ] 核心接口冒烟测试通过
- [ ] 监控告警无异常
- [ ] 日志无 ERROR 级别异常
- [ ] 业务方确认功能正常

---

## 6. 发布后 Checklist

### 6.1 观察期（发布后 30 分钟 - 24 小时）

- [ ] 监控面板持续观察
- [ ] 核心业务指标对比（发布前 vs 发布后）
- [ ] 错误日志无异常增长
- [ ] 慢查询无异常
- [ ] 确认无容量问题

### 6.2 发布确认与复盘

- [ ] 发布结果通知（成功/失败/回滚）
- [ ] 更新变更记录文档
- [ ] 归档发布产物（包/镜像 Tag）
- [ ] 如有问题，发起事故复盘（Postmortem）
