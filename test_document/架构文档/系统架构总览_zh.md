# 系统架构总览

## 1. 文档目标

本文档定义保险公司技术架构的整体视图和设计原则，为所有子系统、技术选型和架构决策提供统一框架。适用于架构师、技术经理和开发人员。

## 2. 架构设计原则

| 原则 | 说明 | 参考来源 |
|------|------|----------|
| 高可用 | 核心系统可用性 ≥ 99.99%，全年停机 < 53 分钟 | Ant Cloud 标准 |
| 容灾多活 | 至少两地三中心部署，关键链路异地多活 | 行业最佳实践 |
| 分布式微服务 | 按业务领域拆分，独立部署、独立扩缩 | Spring Cloud / K8s |
| 松耦合 | 领域间通过异步消息 + 标准 API 交互，避免直连 DB | DDD + CQRS |
| 可观测 | 全链路 Trace、Metrics、Logging 三支柱 | OpenTelemetry |
| 安全合规 | 满足等保 2.0 / 个人信息保护法 / 偿二代 | 监管要求 |

## 4. 核心系统清单

| 系统 | 职责 | 关键能力 |
|------|------|----------|
| 产品中心 | 产品定义、费率因子、条款管理 | 规则引擎、定价模型 |
| 承保中心 | 投保录入、核保决策、出单 | 核保规则引擎、影像OCR |
| 保全中心 | 保单变更、复效、退保 | 保全规则、资金计算 |
| 理赔中心 | 报案、查勘、定损、理算、结案 | 风控模型、反欺诈引擎 |
| 续期中心 | 续期缴费提醒、宽限期管理 | 收付费接口、短信通知 |
| 再保中心 | 临分、合约分入分出 | 合同管理、账单结算 |
| 渠道中心 | 代理人、经代、银保、网销 | 佣金规则、合作协议 |
| 收付中心 | 保费收取、赔付支付、对账 | 资金通道、差错处理 |
| 统一用户 | 客户信息、保单关系、实名认证 | 客户360视图 |
| 数据平台 | 数仓、报表、监管报送、经营分析 | ETL、指标管理 |

## 5. 关键业务流程链路

### 5.1 承保主链路
```
产品销售 → 投保单录入 → 核保自动/人工 → 缴费 → 出单 → 保单生效
```
- 系统间依赖：产品中心 → 承保中心 → 核保(规则/影像) → 收付中心 → 出单
- 非功能要求：高峰期(开门红) TPS ≥ 2000，P99 < 500ms

### 5.2 理赔主链路
```
客户报案 → 查勘 → 立案 → 定损 → 理算 → 核赔 → 结案 → 支付
```
- 系统间依赖：理赔中心 → 核赔规则引擎 → 收付中心 → 再保(如超自留额)
- 非功能要求：单案件处理完成 ≤ 3日（小额快赔 ≤ 1日）

## 6. 架构治理机制

- **架构评审委员会**：每月一次，评审跨域设计方案和技术选型
- **ADR**：所有架构决策记录在仓库中，包含上下文、选项、决策和后果
- **技术雷达**：每季度更新技术栈推荐/退役状态
- **架构合规检查**：CI流水线中集成架构规则扫描（如依赖方向、API契约）
